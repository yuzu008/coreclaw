# レポート生成スキル（Report Writing）

> ピラミッド原則に基づいた構造的なレポートを生成します。

## 概要

Deep Research で収集した情報とフレームワーク分析の結果を統合し、読み手に最適化されたレポートを生成します。
結論先行のピラミッド原則に従い、エグゼクティブサマリーから詳細分析へと展開します。

## レポート構造（ピラミッド原則）

```
1. エグゼクティブサマリー（結論先行）
   - 目的と背景
   - 主要な発見（3つ以内）
   - 推奨アクション

2. 調査概要
   - 調査範囲と手法
   - ソース一覧と信頼度

3. 分析結果
   - フレームワーク分析（詳細）
   - データと根拠
   - インサイト

4. 推奨事項
   - 短期アクション
   - 中長期アクション
   - リスクと対策

5. 付録
   - 詳細データ
   - 参考文献・引用一覧
```

## 品質チェック

レポート完成後、以下の品質チェックを実行します。

### 1. 数値整合性チェック

| チェック項目 | 検証方法 | 不合格条件 |
|------------|---------|----------|
| 数値の一貫性 | 同一数値が本文中で一致 | 矛盾する数値 |
| 計算の正確性 | 合計・割合の再計算 | 計算ミス |
| 単位の統一 | 通貨・単位の確認 | 単位の混在 |

### 2. 引用完全性チェック

| チェック項目 | 検証方法 | 不合格条件 |
|------------|---------|----------|
| ソースの存在 | 全データにURL付記 | 出典なしのデータ |
| リンクの有効性 | URL形式の確認 | 不正なURL |
| 引用の正確性 | ソースとの照合 | 内容の歪曲 |

### 3. コンテンツ整合性チェック

| チェック項目 | 検証方法 | 不合格条件 |
|------------|---------|----------|
| サマリーと本文の一致 | エグゼクティブサマリーの内容確認 | 本文にない結論 |
| MECE網羅性 | 分析の網羅性確認 | 論理的ギャップ |
| 論理の飛躍 | So What/Why So確認 | 根拠のない結論 |

### 4. 形式・構造チェック

| チェック項目 | 検証方法 | 不合格条件 |
|------------|---------|----------|
| テンプレート準拠 | 必須セクションの存在 | セクション欠落 |
| 見出し階層 | H1→H2→H3の構造 | 階層の飛び |
| 表の整形 | テーブルの正しさ | 崩れた表 |

## ハルシネーション防止

- Deep Research のソースで確認できないデータは使用しない
- 推測や推論は「分析に基づく推測」と明記する
- 数値データは必ずソースとの照合を行う
- 引用は原文に忠実に行う（意訳の場合は明記）
