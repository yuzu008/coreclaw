# 深層リサーチスキル（Deep Research）

> Think→Action→Report サイクルで反復的に情報を収集・統合します。

## 概要

仮説駆動の反復リサーチエンジンです。各ラウンドで仮説を生成（Think）、情報を収集（Action）、レポートを更新（Report）するサイクルを繰り返し、網羅的かつ信頼性の高い情報を収集します。

## Think→Action→Report サイクル

```
    ┌─────────────────────────────┐
    │         ラウンド開始          │
    └──────────────┬──────────────┘
                   ↓
    ┌─────────────────────────────┐
    │  🧠 Think: 仮説生成          │
    │  ・知識ギャップの分析          │
    │  ・次に調べるべきことの特定     │
    └──────────────┬──────────────┘
                   ↓
    ┌─────────────────────────────┐
    │  ⚡ Action: 情報収集          │
    │  ・SEARCH: Web検索           │
    │  ・VISIT: ページ詳細取得       │
    │  ・VERIFY: 交差検証           │
    └──────────────┬──────────────┘
                   ↓
    ┌─────────────────────────────┐
    │  📄 Report: レポート更新       │
    │  ・収集情報の統合              │
    │  ・ソース信頼度評価            │
    │  ・次ラウンドの判断            │
    └──────────────┬──────────────┘
                   ↓
         十分な情報か？
        ├── NO → ラウンド開始に戻る
        └── YES → COMPLETE
```

## Phase 1: Think（思考）

各ラウンドの開始時に、以下を分析します。

| 分析項目 | 説明 |
|---------|------|
| 知識ギャップ | 目的達成に対して不足している情報 |
| 仮説 | 現時点で推測される結論 |
| 検証すべき点 | 仮説の裏付けに必要なデータ |
| 次のアクション | SEARCH / VISIT / VERIFY / COMPLETE |

## Phase 2: Action（実行）

| アクション | 用途 | 判断基準 |
|-----------|------|---------|
| **SEARCH** | 新情報が必要 | 知識ギャップが大きい |
| **VISIT** | 詳細取得が必要 | 有望なURLを発見 |
| **VERIFY** | 交差検証 | 重要な数値・事実を複数ソースで確認 |
| **COMPLETE** | 完了 | 目的に対して十分な情報を収集 |

### 検索ルール

- **日英並列検索**: 日本語と英語の両方で検索する
- **アカデミックソース優先**: `.ac.jp`, `.edu`, `.gov` ドメインを優先
- **複数ソース交差検証**: 重要なデータは2件以上のソースで確認

## Phase 3: Report（レポート更新）

各ラウンド後にレポートを更新します。

```
## 現時点の知見

### 確認済みファクト
- [ファクト1] (ソース: URL, 信頼度: 高)
- [ファクト2] (ソース: URL, 信頼度: 中)

### 未確認の仮説
- [仮説1] - 次ラウンドで検証予定
- [仮説2] - 追加情報が必要

### ソース一覧
| # | URL | 種別 | 信頼度 | 取得日 |
|---|-----|------|--------|-------|
| 1 | ... | 学術 | 高 | ... |
| 2 | ... | 企業 | 中 | ... |
```

## ソース品質基準

| 信頼度 | ソース種別 | 例 |
|--------|----------|-----|
| 高 | 政府・学術機関 | 官公庁白書、査読論文、大学研究 |
| 高 | 大手調査機関 | Gartner、IDC、McKinsey |
| 中 | 業界メディア | TechCrunch、日経、Bloomberg |
| 中 | 企業公式 | IR資料、プレスリリース |
| 低 | 個人ブログ・SNS | 参考程度、他ソースで裏付け必要 |

## 完了判定

以下のすべてを満たした場合にCOMPLETEとします。

- [ ] 目的に対して十分な情報が収集されている
- [ ] 主要な仮説が検証されている
- [ ] 3件以上の信頼できるソースがある
- [ ] 重要な数値データが交差検証されている
- [ ] 信頼度の加重平均が60%以上

## ハルシネーション防止

- すべてのファクトにソースURLを付記する
- ソースで確認できないデータは「未確認」と明記する
- 推測・推論は「分析」として明示的に区別する
- 数値データは必ずソースと照合する
