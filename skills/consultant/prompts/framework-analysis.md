# フレームワーク分析スキル（Framework Analysis）

> 53のコンサルティングフレームワークを使って収集情報を構造化分析します。

## 概要

Deep Research で収集した情報を、適切なコンサルティングフレームワークに当てはめて分析します。
MECE（漏れなくダブりなく）の原則で網羅性を検証し、ピラミッド構造で結論を論理的に構造化します。

## ワークフロー

```
1. フレームワーク自動推薦（目的・キーワードから選定）
   ↓
2. フレームワーク適用（収集データを構造化）
   ↓
3. MECE検証（網羅性・排他性チェック）
   ↓
4. ピラミッド構造化（結論→根拠の階層化）
   ↓
5. インサイト導出（分析からの洞察）
   ↓
6. アクション推奨（具体的な次のステップ）
```

## フレームワーク選択基準

リサーチの目的とキーワードに基づいて最適なフレームワークを自動推薦します。

| トリガーキーワード | 推奨フレームワーク |
|------------------|------------------|
| 戦略、強み、弱み、機会、脅威 | SWOT, PEST, 5Forces, 3C |
| マッチング、適合、提携、連携 | マッチングマトリクス |
| ロードマップ、計画、フェーズ | テクノロジーロードマップ |
| 顧客、ユーザー、ペルソナ | ペルソナ分析, STP, カスタマージャーニー |
| 価格、コスト、価値 | 4P, 価値ベースプライシング |
| 市場参入、GTM、ローンチ | AARRR, アンゾフマトリクス |
| 競合、差別化、ポジショニング | ブルーオーシャン, ポジショニングマップ |
| プロセス、効率、最適化 | バリューチェーン, ECRS |
| 意思決定、優先順位、評価 | 意思決定マトリクス, 重み付けスコアリング |
| ビジネスモデル、収益、マネタイズ | BMC, リーンキャンバス |
| 根本原因、問題、なぜ | フィッシュボーン, 5 Whys, イシューツリー |
| 組織、チーム、構造、役割 | 7S, RACI |
| リスク、不確実性、軽減 | リスクマトリクス, シナリオプランニング |
| イノベーション、アイデア | デザイン思考, SCAMPER |
| 成長、拡大、スケール | BCGマトリクス, GEマトリクス |

**重要**: キーワードだけでなく、リサーチの**目的**に基づいてフレームワークを選択すること。
複数のフレームワークが適用可能な場合は以下を優先：
1. ユーザーの核心的な質問に直接回答するフレームワーク
2. Deep Research で収集したデータが利用可能なフレームワーク
3. 深度が同程度の場合、シンプルなフレームワーク

## MECE検証

分析結果をMECE原則で検証します。

```
MECE検証チェックリスト:
□ 相互排他性: 各要素が重複していないか
□ 網羅性: 重要な要素が漏れていないか
□ 粒度の統一: 各要素の抽象度が揃っているか
□ 論理的一貫性: 分類基準が一貫しているか
```

| 検証結果 | 対応 |
|---------|------|
| MECE適合 | そのまま次のステップに進む |
| 重複あり | 要素を統合・再分類する |
| 漏れあり | 欠落している要素を追加する |
| 粒度不統一 | 分解・統合で粒度を揃える |

## ピラミッド構造

分析結果を以下のピラミッド構造で整理します。

```
        ┌─────────────┐
        │   結論       │  ← So What?
        └──────┬──────┘
    ┌──────────┼──────────┐
    │          │          │
┌───┴───┐ ┌───┴───┐ ┌───┴───┐
│ 根拠1  │ │ 根拠2  │ │ 根拠3  │  ← Why So?
└───┬───┘ └───┬───┘ └───┬───┘
    │         │         │
  [データ]  [データ]  [データ]  ← ファクト
```
