# 目的探索スキル（Purpose Discovery）

> ユーザーの「表層目的」から「真の目的」を探索し、リサーチ計画を立案します。

## 概要

仮説駆動アプローチに基づき、ユーザーとの1問1答対話を通じて真の目的を発見します。
5 Whys分析とJTBD（Jobs To Be Done）分析を組み合わせ、表層的な依頼の裏にある本質的なニーズを特定します。

## ワークフロー

```
1. 構造化プロンプト生成（6要素に分解）
   ↓ ユーザー承認待ち
2. 1問1答対話（最低3ラウンド）
   ↓
3. 5 Whys分析（抽象的回答を掘り下げ）
   ↓
4. JTBD分析（3つのジョブを特定）
   ↓
5. 真の目的を定義
   ↓
6. リサーチ計画を立案
```

## Step 1: 構造化プロンプト

ユーザーのリクエストを以下の6要素に分解して提示します。

| 要素 | 説明 |
|------|------|
| PURPOSE | 何を決定・判断したいか |
| TARGET | 何/誰を調査するか |
| SCOPE | 調査の深さと幅 |
| TIMELINE | 情報の時期・期限 |
| CONSTRAINTS | 調査の制約条件 |
| DELIVERABLES | 期待する成果物 |

**必ず表形式でユーザーに提示し、承認を得てから次に進むこと。**

## Step 2: 質問カテゴリ

1問1答形式で以下のカテゴリから質問します。**複数の質問を一度に投げることは禁止**です。

| カテゴリ | 質問例 | 目的 |
|---------|--------|------|
| **WHY** | 「なぜこの調査が必要ですか？」 | 根本的動機の発見 |
| **WHO** | 「この結果を誰が使いますか？」 | ステークホルダー特定 |
| **WHAT-IF** | 「理想的な結果が得られたら次に何をしますか？」 | 期待するアクション |
| **CONSTRAINT** | 「時間・予算・範囲の制約はありますか？」 | 調査スコープ確定 |
| **SUCCESS** | 「何をもって成功と定義しますか？」 | 完了基準の明確化 |

## Step 3: 5 Whys分析

抽象的な回答には最大5回「なぜ？」で掘り下げます。

```
Why 1: なぜAI市場を調べたいのか？ → 新規事業を検討している
Why 2: なぜ新規事業なのか？ → 既存事業の成長鈍化
Why 3: なぜ成長鈍化なのか？ → 市場飽和と価格競争
Why 4: なぜ価格競争に陥っているのか？ → 差別化要素の欠如
Why 5: なぜ差別化できていないのか？ → 技術的優位性の喪失
→ 真の目的: 技術的差別化による新規事業領域の特定
```

## Step 4: JTBD（Jobs To Be Done）分析

ユーザーが「雇いたい仕事」を3つの観点で特定します。

| ジョブ | 説明 | 質問例 |
|--------|------|--------|
| 機能的ジョブ | 達成したい具体的なタスク | 「この情報で具体的に何をしたいですか？」 |
| 感情的ジョブ | 感じたい/避けたい感情 | 「この調査結果でどう感じたいですか？」 |
| 社会的ジョブ | 他者からどう見られたいか | 「この成果を誰にどう評価されたいですか？」 |

## Step 5: 出力

真の目的を明示し、ユーザーに確認を求めます。

```
【真の目的】
「新規事業としてAI関連サービスを立ち上げるため、
 参入可能なニッチ市場を特定し、
 3ヶ月以内に事業計画書の基礎資料を完成させる」

この目的で合っていますか？修正があればお知らせください。
```

## Step 6: リサーチ計画

確定した目的に基づき、調査計画を立案します。

| 項目 | 内容 |
|------|------|
| 調査テーマ | 真の目的から導出 |
| 調査範囲 | SCOPE + CONSTRAINTS から決定 |
| 検索クエリ | 日本語・英語それぞれのクエリリスト |
| 推奨フレームワーク | 目的に適したフレームワークを推薦 |
| 想定成果物 | DELIVERABLES に基づく |
