# オーケストレーター

> ユーザーのリクエストを分析し、最適なスキルにルーティングする指揮者です。

## 概要

このスキルは、ユーザーの入力を分析し、teaching-assistant パッケージ内の適切なスキルに振り分けます。複合的なリクエストの場合は、複数スキルの実行順序を管理します。

## インテント分析

### キーワードルーティング

| キーワード | スキル | 優先度 |
|-----------|--------|--------|
| 授業計画、指導案、本時の展開 | lesson-plan | 高 |
| ワークシート、スライド、教材、プリント、クイズ | materials | 高 |
| テスト、ルーブリック、評価、採点 | assessment | 高 |
| 個別指導、支援計画、学習スタイル、ZPD | individual | 高 |
| フィードバック、コメント、添削、振り返り | feedback | 高 |
| 生活指導、生徒指導、不登校、いじめ、行動 | guidance | 高 |
| 教育理論、理論、エビデンス | theory-lookup | 中 |

### インテント解析フロー

```
ユーザー入力
  ↓
1. キーワードマッチング
  ↓
2. コンテキスト分析（文脈から意図を推測）
  ↓
3. 曖昧さの解消（必要に応じてユーザーに確認）
  ↓
4. スキル選択・実行
```

### 曖昧なリクエストの処理

キーワードだけでは判断できない場合：

1. **文脈から推測**: 会話の流れから最も適切なスキルを選択
2. **確認を求める**: 複数候補がある場合はユーザーに確認

```
「〜について」というリクエストですが、以下のどちらをご希望ですか？

1. 📋 授業計画を作成する
2. 📝 教材（ワークシート等）を作成する
3. 📊 評価ツールを作成する
```

## マルチスキル連携

### 実行順序

複合リクエスト時の推奨実行順序：

```
meta-prompt（コンテキスト収集）
  ↓
lesson-plan（授業計画）
  ↓
materials（教材作成）
  ↓
assessment（評価設計）
  ↓
feedback（フィードバック）
```

### 依存関係

| スキル | 依存先 | 依存内容 |
|--------|--------|----------|
| materials | lesson-plan | 授業の流れ、時間配分 |
| assessment | lesson-plan | 学習目標、評価観点 |
| feedback | assessment | 評価基準、観点 |
| individual | assessment | 評価結果、課題 |

### 連携パターン例

#### パターン1: 授業準備一式
```
ユーザー: 「中学2年の一次関数の授業を準備したい」

実行:
1. meta-prompt → コンテキスト収集
2. lesson-plan → 授業計画作成
3. materials → ワークシート作成
4. assessment → 形成的評価ツール作成
```

#### パターン2: 評価→フィードバック
```
ユーザー: 「テストの結果にフィードバックを書きたい」

実行:
1. meta-prompt → コンテキスト収集
2. feedback → フィードバック生成
```

#### パターン3: 個別支援計画
```
ユーザー: 「学習が遅れている生徒への指導を考えたい」

実行:
1. meta-prompt → コンテキスト収集
2. individual → 個別指導計画作成
3. materials → 個別教材作成
```

## コンテキスト引き継ぎ

スキル間でコンテキストを引き継ぐ際の規則：

1. **メタプロンプト情報**: すべてのスキルに自動的に渡される
2. **生成コンテンツ**: context-manager を通じて次のスキルに渡す
3. **整合性チェック**: 関連するスキル間で目標・内容の一貫性を確認

## フォールバック処理

教育に関連しないリクエストの場合：

```
申し訳ありませんが、このスキルパッケージは教育・授業支援に特化しています。

以下のような内容でお手伝いできます：
- 📋 授業計画の作成
- 📝 教材（ワークシート・スライド等）の作成
- 📊 評価ツール（ルーブリック・テスト等）の設計
- 👤 個別指導計画の作成
- 💬 学習成果へのフィードバック
- 🏫 生活指導案の作成

何についてお手伝いしましょうか？
```
