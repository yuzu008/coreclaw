# メタプロンプト生成スキル

> ユーザーの教育ニーズを分析し、最適なスキル実行のためのコンテキストを収集するメタプロンプトを生成します。

## 概要

このスキルは、ユーザーのリクエストを分析し、他のスキル（授業計画、教材作成、評価設計等）の実行に必要な情報を効率的に収集します。1回に1つの質問で、最大5問まで聞き取りを行います。

## 3つのフェーズ

### Phase 1: プロンプト強化

ユーザーの入力を分析し、以下を明確化します：

- **目的**: 何を達成したいのか
- **対象**: 誰に向けたものか
- **形式**: どのような成果物が求められるか
- **制約**: 時間・リソース等の制約

### Phase 2: コンテキスト収集

**1回に1つの質問** で、必要な情報を順番に収集します。

#### 収集する情報

| 優先度 | 項目 | 例 |
|--------|------|-----|
| 必須 | 学年 | 中学2年 |
| 必須 | 教科 | 数学 |
| 必須 | 単元/トピック | 一次関数 |
| 推奨 | 学習目標 | 2点を通る直線の式を求められる |
| 推奨 | 時間 | 50分 |
| 任意 | 特別な配慮 | 支援が必要な生徒3名 |

#### 質問ルール

1. **最大5問** まで（不足情報が多くても5問以内にまとめる）
2. **1回に1つ** の質問のみ
3. 選択肢がある場合は提示する
4. デフォルト値がある場合は明示する
5. ユーザーが「おまかせ」と言った場合は適切なデフォルト値を使用

#### 質問テンプレート

```
{前の回答への応答}

次の質問です：

**{質問内容}**

{選択肢がある場合}
- A: {選択肢A}
- B: {選択肢B}
- C: {選択肢C}

{デフォルト値がある場合}
（指定がなければ「{デフォルト値}」で作成します）
```

### Phase 3: メタプロンプト生成

収集した情報を構造化し、実行スキルに渡す形式にまとめます。

#### 生成形式

```markdown
## 収集コンテキスト

### 基本情報
- **学年**: {学年}
- **教科**: {教科}
- **単元/トピック**: {トピック}

### 学習目標
{学習目標の記述}

### 制約条件
- **時間**: {時間}
- **環境**: {教室環境}
- **特別な配慮**: {配慮事項}

### 補足情報
{追加で得た情報}
```

## スキップ条件

以下の場合、質問をスキップして直接実行に移行します：

- ユーザーが十分な情報を最初から提供している場合
- 「おまかせ」「デフォルトで」と指示された場合
- 緊急性が高い場合（明日の授業で使いたい等）

## 実行スキルへの引き継ぎ

収集したコンテキストは、context-manager を通じて以下のスキルに渡されます：

| ユーザーの意図 | 実行スキル |
|---------------|-----------|
| 授業計画を作りたい | lesson-plan |
| 教材を作りたい | materials |
| テスト・評価を作りたい | assessment |
| 個別指導計画を立てたい | individual |
| フィードバックを書きたい | feedback |
| 生活指導案を考えたい | guidance |

## 注意事項

- 質問は**日本語**で行う（ユーザーが英語で質問した場合は英語で対応）
- 教育の専門用語は必要に応じて平易な言葉で補足する
- 質問のトーンは丁寧かつ親しみやすく
