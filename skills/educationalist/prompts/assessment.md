# 評価設計スキル

> Bloom's Taxonomyに基づいた、学習目標と整合性のある評価ツールを生成します。

## 概要

このスキルは、メタプロンプトで収集したコンテキストを基に、構成的整合性（Constructive Alignment）を意識した評価ツールを作成します。

## 入力要件

メタプロンプトから以下の情報を受け取ります：

- **必須**: 学年、教科、単元/トピック、評価タイプ
- **推奨**: 学習目標、評価の目的（形成的/総括的）
- **任意**: 評価観点、配点、フィードバック方針

## 学習指導要領参照

> **小中高の評価設計時は、必ず学習指導要領を参照すること。**
>
> **重要**: 学習指導要領の検索には `data/curriculum.db` (SQLite) を使用してください。
> `data/curriculum/*.md` を grep で検索しないでください（遅いため）。

### 検索方法

```bash
# 教科・単元のキーワードで検索（3文字以上）
sqlite3 data/curriculum.db "SELECT school_level, heading, substr(body,1,500) FROM sections_fts f JOIN sections s ON f.rowid = s.rowid WHERE sections_fts MATCH '{教科名や単元の3文字以上のキーワード}' AND s.school_level = '{学校種}' LIMIT 5;"

# 2文字キーワードの場合
sqlite3 data/curriculum.db "SELECT school_level, heading, substr(body,1,500) FROM sections WHERE school_level = '{学校種}' AND (heading LIKE '%{キーワード}%' OR body LIKE '%{キーワード}%') LIMIT 5;"
```

評価ツールには必ず以下のセクションを含めます：

```markdown
### 学習指導要領との関連

**参照箇所**: {学校種}学習指導要領 第{章}章 第{節}節 {教科} {学年}
**評価の観点**: 知識・技能 / 思考・判断・表現 / 主体的に学習に取り組む態度
**目標との対応**: {評価項目が学習指導要領のどの目標に対応するか}
```

## 対応する評価タイプ

| タイプ | 説明 | 適した場面 |
|--------|------|-----------|
| `rubric` | ルーブリック評価 | パフォーマンス評価、作品評価 |
| `test` | テスト問題 | 知識・理解の確認 |
| `formative` | 形成的評価 | 学習途中の確認・指導改善 |

```markdown
## フィードバック方針

### 即時フィードバック

## 参照した教育理論
- **Formative Assessment (Black & Wiliam)**: 学習のための評価
- **Feedback Loops**: 継続的な改善サイクル
```

## 構成的整合性（Constructive Alignment）

John Biggs の構成的整合性理論に基づき、以下を確認：

```
学習目標 ←→ 指導活動 ←→ 評価
```

### 整合性チェックリスト

- [ ] 学習目標と評価項目が対応している
- [ ] 目標の認知レベルと評価の難易度が一致している
- [ ] 指導活動で練習した内容が評価される
- [ ] 評価基準が明確で、生徒にも共有可能

## 認知レベル別問題設計

### Bloom's Taxonomy 適用ガイド

| レベル | 問題タイプ | 動詞例 |
|--------|-----------|--------|
| 記憶 | 用語の定義、事実の再生 | 述べよ、列挙せよ |
| 理解 | 説明、要約、言い換え | 説明せよ、比較せよ |
| 応用 | 問題解決、手順の実行 | 計算せよ、作成せよ |
| 分析 | 関係の分析、パターン発見 | 分析せよ、区別せよ |
| 評価 | 批評、判断、正当化 | 評価せよ、判断せよ |
| 創造 | 新しいものの創出、設計 | 設計せよ、提案せよ |

## 主要な評価理論

| 理論 | 用途 |
|------|------|
| Constructive Alignment | 目標・指導・評価の一貫性 |
| Bloom's Taxonomy | 認知レベル別の問題設計 |
| Formative Assessment | 学習過程のフィードバック |

## 次のステップ

評価設計後、以下の連携が可能です：

- `feedback` → 評価結果に基づくフィードバックを生成
- `individual` → 評価結果から個別支援計画を作成
