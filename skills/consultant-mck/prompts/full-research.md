# McKinsey 統合リサーチ・プロンプト

## 概要

マッキンゼー式5フェーズで深層リサーチを実行する統合ワークフローです。
仮説駆動・MECE・ピラミッド原則を中核に、構造的な分析と提言を行います。

---

## ワークフロー

### Phase 1: 問題定義（Problem Definition）

```
1. クライアントの表層的な課題を聞く
2. 「なぜそれが問題なのか？」を5回掘り下げる
3. 真の課題（Root Issue）を特定する
4. 成功の定義を明確化する
   - 定量的成功基準: [KPI、数値目標]
   - 定性的成功基準: [状態目標]
   - 時間軸: [いつまでに]
5. 分析のスコープを設定する
   - 対象範囲: [地域/事業/部門]
   - 除外範囲: [分析対象外]
```

### Phase 2: 仮説構築（Hypothesis Building）

```
1. Day 1 Answer を設定する
   「もし今日答えを出すなら、それは___である」
2. 支持論拠を3つ以上列挙する
3. 各論拠の反証条件を明記する
   「もし___が事実であれば、この仮説は棄却される」
4. 仮説の強度を評価する
   - 確信度: [1-10]
   - 最大リスク: [仮説が誤りだった場合の影響]
```

### Phase 3: イシュー分解（Issue Decomposition）

```
1. Key Questionをイシューツリーに分解する
2. MECE検証を行う
   - [ ] Level 1 でトップダウンに全体をカバー
   - [ ] 各レベルで重複なし
   - [ ] 末端は検証可能な粒度
3. 各イシューにインパクトスコアを付与する
   - インパクト（事業への影響）: High/Med/Low
   - 検証容易性（データ入手性）: High/Med/Low
4. 80/20ルールで最重要イシューを特定する
   → 上位20%のイシューにフォーカス
5. ワークプランを策定する
   | イシュー | 分析手法 | データソース | 担当 | 期限 |
```

### Phase 4: データ収集・分析（Data Collection & Analysis）

```
1. データソースを特定し収集する
   - 一次情報: インタビュー / 調査 / 実験
   - 二次情報: レポート / 統計 / 公開データ
2. 適切なフレームワークで分析する
   （7S / 3つの地平線 / OVA / プロフィットプール等）
3. 各分析結果に「So What?」を問う
   - このデータは何を意味するか？
   - 経営判断にどう影響するか？
   - 仮説を支持するか、棄却するか？
4. 仮説を更新する
   - 支持されたポイント: [列挙]
   - 修正が必要なポイント: [列挙]
   - 新たに発見された論点: [列挙]
```

### Phase 5: 統合・提言（Synthesis & Recommendation）

```
1. ピラミッド原則で最終提言を構造化する
   ┌─ 中心的結論（1文）
   ├── 根拠A（主張＋ファクト＋So What）
   ├── 根拠B（主張＋ファクト＋So What）
   └── 根拠C（主張＋ファクト＋So What）

2. 提言の実行プランを策定する
   | アクション | 責任者 | 期限 | 必要リソース | 期待効果 |

3. リスク分析
   | リスク | 発生確率 | 影響度 | 対策 |

4. クイックウィン（即座に着手可能な施策）
   - 30日以内: [施策1]
   - 90日以内: [施策2]
   - 180日以内: [施策3]

5. 成功の測定方法
   | KPI | 現在値 | 目標値 | 測定頻度 |
```

---

## フレームワーク選択ガイド

| 課題タイプ | 推奨フレームワーク | Phase |
|-----------|------------------|-------|
| 組織変革 | 7S | Phase 4 |
| 成長戦略 | 3つの地平線 + グラニュラリティ | Phase 4 |
| コスト最適化 | OVA | Phase 4 |
| 業界構造理解 | SCP + プロフィットプール | Phase 4 |
| 問題の構造化 | イシューツリー | Phase 3 |
| 報告の構造化 | ピラミッド原則 | Phase 5 |

## 品質チェックリスト

- [ ] Day 1 Answer は設定済みか
- [ ] イシューツリーはMECEか
- [ ] 80/20 で重要イシューに集中しているか
- [ ] 全分析にSo What?があるか
- [ ] ファクトベースの分析か（意見vs事実を区別）
- [ ] ピラミッド構造で結論→根拠が明確か
- [ ] アクションプランは具体的で実行可能か
- [ ] クイックウィンが特定されているか
