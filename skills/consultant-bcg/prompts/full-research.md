# BCG統合リサーチ

> BCGメソドロジーで4フェーズを一貫実行する統合リサーチワークフローです。

## MCP連携

`deep-research` MCP サーバーが利用可能な場合は、Phase 2（Deep Research）で MCP の
`deep-research` プロンプトテンプレートを活用してください。MCP が研究課題の精緻化、
サブ質問への分解、Web検索、ソース評価、レポート生成の構造化フレームワークを提供します。
MCP が利用できない場合は従来のワークフローを使用します。

## 実行フロー

```
Phase 1: Purpose Discovery
 │  ・BCG Hypothesis-Driven Approach で Key Question 設定
 │  ・Day 1 Answer（初期仮説）設定
 │  ・Issue Tree でMECE分解
 │
Phase 2: Deep Research
 │  ・Think→Action→Report サイクル
 │  ・仮説検証のためのファクト収集
 │
Phase 3: BCG Framework Analysis
 │  ・Growth-Share Matrix / Experience Curve / Strategy Palette 等
 │  ・MECE検証、So What分析
 │
Phase 4: Report Writing
 │  ・ファクトベース、ピラミッド構造
 │  ・品質チェック → ユーザー承認
```

## BCG特有の分析アプローチ

### ポートフォリオ案件

1. Hypothesis-Driven で初期仮説設定
2. Issue Tree でイシューMECE分解
3. Growth-Share Matrix でポートフォリオ可視化
4. Advantage Matrix で業界特性把握
5. Strategy Palette で戦略アプローチ選択
6. TSR Analysis で価値創造定量化

### コスト変革案件

1. Experience Curve でコストポジション把握
2. Deconstruction でバリューチェーン分析
3. Time-Based Competition で時間効率改善
