# 研究者向けアプリ構想（完全保存版）

作成日: 2026-03-24
目的: 直近の会話で提示したアイディアを、削らずにそのまま保存する。

## 0. 位置づけ
この構想は「教育カリキュラム作成アプリ」ではなく、研究設計から実験・論文までを一気通貫で支援する研究ワークベンチとして設計する。

## 1. 研究者向けでの設計軸
1. 再現性第一
- プロンプト、データ版、コード版、結果を必ず紐づけて保存する。

2. 実験管理第一
- 会話より Experiment Run を主役にする。

3. 根拠付き生成
- 生成結果には出典、使用データ、推論手順を必須添付する。

4. 査読対応
- 図表・結果・設定をそのまま再生成できる形で残す。

## 2. 機能の主役（大学研究者向け）
1. 研究プロジェクト管理
- 課題、仮説、データ、実験計画、倫理審査メモを管理。

2. 実験トラッキング
- 各実行の条件、モデル、パラメータ、成果物を自動記録。

3. 文献・知識統合
- 関連論文の要約、比較表、引用候補の生成。

4. 解析と可視化
- 統計解析、図表生成、アブレーション比較。

5. 論文執筆支援
- Method、Results、Limitations の下書きと図表キャプション生成。

6. 共同研究対応
- 誰が何を変更したか、レビューコメント、差分比較。

## 3. 最小MVP（研究者向け）
1. Project 作成
2. Run 実行と履歴保存
3. 成果物自動保存（図、表、ログ、設定）
4. 再実行ボタン（同条件再現）
5. 研究メモと論文下書き出力

## 4. 成功指標
1. 再現実験成功率
2. 1本の実験報告作成時間
3. 論文図表作成時間
4. 共同研究での手戻り減少

## 5. 実装の考え方（補助メモ）
- 生成AIアプリとしてではなく、研究ワークフローアプリとして設計する。
- AIは「案を出す」、研究者は「採択・編集・確定する」を明確に分離する。
- 文章品質ではなく、準備時間短縮・改善速度・再現性で価値を測る。
