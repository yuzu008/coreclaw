# CoreClaw デモ用プロンプト集

各機能別に10個のデモンストレーション用プロンプトをまとめました。
デモ実施時に、シナリオに合わせてこれらのプロンプトを投入してください。

---

## 1. Scientific Research（科学研究支援）

基礎研究、データ分析、論文作成などの研究ワークフローを支援するプロンプト。

1. **遺伝子発現データの統計分析**
   ```
   マウスの脳組織から取得した遺伝子発現データ（CSV形式、200遺伝子×50サンプル）があります。
   コントロール群とTreatment群で発現量に有意な差があるか、
   Welchのt検定と多重比較補正を用いて分析してください。
   有意なトップ10遺伝子の火山図を作成し、パターンを解釈してください。
   ```

2. **タンパク質構造予測と分子動力学シミュレーション**
   ```
   新規医薬品候補化合物のタンパク質標的との相互作用を予測したいです。
   タンパク質の PDB ID は 1ABC です。
   リガンドドッキング、相互作用エネルギーの計算、MD シミュレーション 100ns を実行し、
   バインディングモード安定性を評価するレポートを作成してください。
   ```

3. **多次元データセットの次元削減と可視化**
   ```
   患者の臨床検査データ（40項目、500患者）があります。
   PCA、t-SNE、UMAP を用いて次元を削減し、
   病態分類（正常/疾患A/疾患B）との関連を可視化してください。
   結果の解釈と次元削減手法の比較分析を含めてください。
   ```

4. **メタゲノム解析：マイクロバイオーム構成分析**
   ```
   腸内マイクロバイオーム 16S rRNA の塩基配列データ（FASTQ形式）があります。
   分類学的多様性（Alpha/Beta diversity）を計算し、
   OTU/ASV レベルで群間の相違を検定してください。
   プロット、統計表、生物学的解釈を含めたレポートを生成してください。
   ```

5. **時系列生物学的データの周期性検出**
   ```
   24時間注視下での患者の体温・心拍・活動量のウェアラブルセンサーデータがあります。
   サーカディアンリズムと準周期成分を検出・分離し、
   個人のクロノタイプ（朝型/夜型）を判定するアルゴリズムを実装してください。
   ```

6. **論文から重要な知見を抽出・比較表作成**
   ```
   同一テーマの査読済み論文5本をアップロードしました。
   各論文の研究方法、主要な知見、統計結果、限界をまとめた比較表を作成してください。
   メタアナリシスの可能性と研究ギャップを指摘するレビューを生成してください。
   ```

7. **統計検定の選択・実施・結果報告**
   ```
   複数群のベースライン特性データがあります。
   群間比較に適切な検定（パラメトリック/ノンパラメトリック）を選択し、
   effect size、信頼区間、p値を計算してください。
   結果の臨床的意義と表現方法を含む標準的なレポートを生成してください。
   ```

8. **機械学習モデルの開発・評価・解釈**
   ```
   患者の初診時データから 1年以内の再入院を予測するモデルを開発してください。
   特徴選択、複数アルゴリズムの比較（ロジスティック回帰、ランダムフォレスト、XGBoost）、
   クロスバリデーション、SHAP 値による予測解釈を実施してください。
   ```

9. **生物学的データベース検索・文献マイニング**
   ```
   「KRAS 遺伝子変異」と「膵がん予後」に関する最新の生物学的知見を調査してください。
   PubMed、DrugBank、COSMIC データベースから関連情報を集約し、
   臨床応用可能性を検討するシステマティックレビューを作成してください。
   ```

10. **研究論文画像の再現・チェック・改善提案**
    ```
    投稿予定の論文の Figure 1 〜 3（PNG形式）があります。
    図の見栄え、凡例、色使い、キャプションの正確さをレビューしてください。
    投稿ジャーナル（Nature Biomedical Engineering 想定）のガイドラインに準拠した
    改善版を提案・生成してください。
    ```

## 2. MUSUBIx / Specification-Driven Development（要件・設計・実装トレーサビリティ）

ソフトウェア開発の要件定義から実装までの構造化ワークフロー。

1. **新機能の要件定義（EARS形式）**
   ```
   CoreClaw に「実験結果の自動要約・キーワード抽出機能」を追加したいです。
   EARS形式で機能要件を定義してください。
   以下を含めてください：前提条件、入出力、制約、例外ケース、受入条件。
   ```

2. **C4 モデルによる設計書作成**
   ```
   Web サーバー + Docker コンテナ + MCP ツール連携システムの
   C4 Context、Container、Component レベルの設計図を生成してください。
   コンポーネント間の責務分離と通信プロトコルを明示するドキュメントを作成してください。
   ```

3. **トレーサビリティマトリックス作成**
   ```
   以下の要件に対して、設計・実装・テストケースのトレーサビリティマトリックスを作成してください：
   - REQ-001: 実験メタデータの自動入力
   - REQ-002: 実験結果の版管理
   - REQ-003: GitHub への自動同期
   
   各要件の設計ID、コンポーネント、テストケースを紐付けてください。
   ```

4. **タスク分解と実装計画**
   ```
   以下の機能を実装するための Work Breakdown Structure (WBS) を作成してください：
   「MCP ツール管理ダッシュボード」
   
   各タスクに TSK-* ID を付与し、依存関係、工数見積、リスク、検証方法を記載してください。
   ```

5. **アーキテクチャ決定記録（ADR）**
   ```
   CoreClaw で次の決定をしました：
   - Docker イメージのビルド戦略: マルチステージビルド vs. キャッシング
   - 認証方式: GitHub OAuth vs. API トークン
   - データベース: SQLite vs. PostgreSQL
   
   各決定について、背景、検討案、選択理由、トレードオフを記載した ADR を生成してください。
   ```

6. **品質ゲート定義と検証チェックリスト**
   ```
   新しいスキル追加時の品質ゲートを定義してください：
   - 要件フェーズ承認条件
   - 設計ドキュメント完成度
   - コード品質メトリクス（カバレッジ、複雑度など）
   - テスト合格条件
   
   各ゲートの検証チェックリストを作成してください。
   ```

7. **設計レビュー指摘事項の起票・対応**
   ```
   設計ドキュメントのレビュー結果：
   - 指摘1: WebSocket 接続失敗時の再試行ロジックが未定義
   - 指摘2: MCP ツール間のデータモデル不統一
   - 指摘3: エラーハンドリング基準が曖昧
   
   各指摘について、対応タスク化、影響分析、修正計画を作成してください。
   ```

8. **テスト戦略策定と テストケース生成**
   ```
   「実験管理機能」（作成・編集・削除・検索・エクスポート）のテスト戦略を策定してください。
   ユニット、統合、E2E、セキュリティテストを覆う 30個のテストケースを
   テンプレート形式で生成してください。
   ```

9. **コード・設計間の乖離分析**
   ```
   実装済みコード（src/ フォルダ）と設計書の齟齬を分析してください。
   実装が設計から逸脱している箇所、実装にあるが設計に無い機能、
   ドキュメント化されていない制約・パターンを列挙してください。
   ```

10. **リリースノート・マイグレーション計画**
    ```
    CoreClaw v0.2.0 → v0.3.0 への更新内容（新機能 5個、バグ修正 8個）をもとに、
    ユーザー向けリリースノート、システム管理者向けマイグレーション手順、
    ロールバック計画を作成してください。
    ```

## 3. Experiment Management（実験管理・再現性）

実験の計画、実行、記録、再現をサポートするプロンプト。

1. **実験計画書の生成**
   ```
   テーマ：「遺伝子ノックアウトマウスの認知行動試験」
   背景、目的、仮説、研究計画（被験者数、処理群、測定項目）、
   統計解析計画、倫理上の配慮を含む実験計画書を生成してください。
   ```

2. **実験条件・パラメータの版管理記録**
   ```
   同一実験を異なるパラメータで 3回繰り返す予定です。
   各回の条件（試薬ロット番号、温度、時間など）を構造化して記録・比較するテンプレートを作成してください。
   ```

3. **実験結果の再現性確認チェック**
   ```
   過去10日間に実施した 5個の実験結果があります。
   同じプロトコルを再度実施したとき、どのように再現性を判定すべきか、
   判定基準と検証方法を定義するガイドを作成してください。
   ```

4. **実験から論文図表への変換フロー**
   ```
   実験 1 で生成されたデータセット・プロット・統計値があります。
   これを論文に掲載可能な形式（高解像度 PNG、LaTeX 対応表、キャプション）に
   自動変換し、原稿ファイルに埋め込むワークフローを設計してください。
   ```

5. **複数実験者による同一プロトコルの結果比較**
   ```
   4名の異なる実験者が同一プロトコルで実験を実施しました。
   結果のばらつきを分析し、群間で有意な差異が無いか確認するリポートを作成してください。
   Validation / 実験者間再現性の定量的評価を含めてください。
   ```

6. **長期的な実験シリーズのデータマネジメント**
   ```
   6ヶ月間連続で毎週実施される実験（52回）のデータを一元管理するシステムを設計してください。
   メタデータスキーマ、バージョン管理、品質監査の仕組みを含めてください。
   ```

7. **失敗実験の問題分析・改善計画**
   ```
   期待した結果が得られなかった実験があります。
   失敗原因の仮説的分類（サンプル品質、手技エラー、機器不具合、統計偶然など）、
   原因特定の追加実験、改善対策を整理するテンプレートを作成してください。
   ```

8. **外部委託実験の品質管理**
   ```
   外部の受託分析機関で実施された遺伝子解析結果を受け取りました。
   データの完全性、統計的有意性、施設の認証状況を確認・文書化するチェックリストを作成してください。
   ```

9. **実験系統図・フローの可視化**
   ```
   複雑な実験シリーズ（前実験 → 本実験 → 追加実験）を、意思決定ポイント、
   並行実験、条件分岐を含むフローチャートで可視化してください。
   Mermaid / PlantUML で自動生成するコードも提供してください。
   ```

10. **実験ノート・ラボノートの自動サマリー**
    ```
    1か月分の実験ノート（日付、実施内容、結果、気付き）がテキストで記録されています。
    週単位・月単位の要約レポート、重要な知見抽出、問題事項の振り返り、
    次期実験への提言を自動生成してください。
    ```

## 4. Skills & Tools Integration（スキル・ツール統合・MCP）

MCP ツール、カスタムスキル、外部 API 連携。

1. **カスタム MCP サーバーの設計・実装**
   ```
   CoreClaw に「文献管理システム MCP」を統合したいです。
   PubMed API、arXiv API、Zotero API へのアクセス仲介、
   引用形式の自動変換、バイオメトリクスの取得を実現する MCP インターフェースを設計してください。
   ```

2. **MCP ツール間のデータモデル設計**
   ```
   複数の MCP ツール（GitHub MCP、Deep Research MCP、カスタム Database MCP）から
   返されるデータを統一的に扱うためのスキーマを設計してください。
   型定義、変換ロジック、エラーハンドリング戦略を含めてください。
   ```

3. **スキル権限管理・ホワイトリスト設定**
   ```
   195個の Scientist スキルのうち、セキュリティリスク評価を実施して、
   クラウド環境・オンプレミス環境別のホワイトリストを定義してください。
   各スキルの許可/禁止理由と使用条件を記載してください。
   ```

4. **新しいスキルの追加検証フロー**
   ```
   新規スキル「臨床試験設計アシスタント」を追加したいです。
   スキルの読み込み、マニフェスト検証、セキュリティスキャン、テスト実行、
   本番環境への適用までの完全なワークフローを策定してください。
   ```

5. **MCP ツール呼び出しのパフォーマンス最適化**
   ```
   複数の MCP ツールを順序有りで呼び出す際、
   ネットワーク遅延とタイムアウトのリスクが高まっています。
   キャッシング、並列実行、フォールバック戦略を含む設計を提案してください。
   ```

6. **外部 API の認証・トークン管理**
   ```
   GitHub、PubMed、AWS、Azure のトークン/クレデンシャルを安全に管理し、
   複数ユーザー間で共有するシステムを設計してください。
   暗号化、ローテーション、監査ログを含めてください。
   ```

7. **スキル使用ログの分析・レポート**
   ```
   過去 30日間のスキル使用ログ（どのスキルが、いつ、何回使用されたか）を分析して、
   利用パターン、人気スキル排序、未使用スキル、エラー発生箇所をレポートしてください。
   ```

8. **カスタムスキルのドキュメンテーション・チュートリアル生成**
   ```
   開発済みのカスタムスキル（SKILL.md、サンプルコード有）をもとに、
   新規ユーザー向けのチュートリアル、FAQ、トラブルシューティングガイドを自動生成してください。
   ```

9. **スキル互換性マトリックス**
   ```
   各スキルが依存している外部ライブラリ、Python バージョン、メモリ要件を調査して、
   スキル間の互換性マトリックスを作成してください。
   与えられた環境で実行可能なスキル組み合わせを提示してください。
   ```

10. **MCP ツール仕様ドキュメント生成**
    ```
    自社開発の MCP ツール（ソースコード提供）の機能、API エンドポイント、
    入出力仕様、エラーコード、使用例を含む OpenAPI 互換の仕様書を自動生成してください。
    ```

## 5. GitHub Integration（GitHub 連携・リポジトリ同期）

実験結果・コード・ドキュメントの GitHub への自動同期・管理。

1. **実験結果の GitHub への自動プッシュ**
   ```
   実験完了時に、結果ファイル（図表、CSV、JSON）を自動的に GitHub リポジトリにプッシュするワークフローを設計してください。
   コミットメッセージの自動生成、タグ付け、PR 作成の自動化を含めてください。
   ```

2. **GitHub Issues からの実験タスク自動生成**
   ```
   GitHub Issues で「実験を実施する」と記載されたタスクを監視して、
   CoreClaw の実験キューに自動で登録するシステムを設計してください。
   ステータス同期、自動クローズ機能を含めてください。
   ```

3. **Pull Request レビューフロー**
   ```
   新しいスキルやコード変更を Pull Request で提案するワークフローを設計してください。
   自動テスト、コード品質チェック、セキュリティスキャン、レビュアー自動割り当てを含めてください。
   ```

4. **リポジトリからの設定・スキル自動取得**
   ```
   外部の GitHub リポジトリから、最新のスキル定義、設定ファイル、ドキュメントを
   定期的に自動取得・マージするシステムを実装してください。
   コンフリクト解決、バージョン管理を含めてください。
   ```

5. **実験結果の GitHub Releases への公開**
   ```
   重要な実験が完了したとき、結果をデータセットして GitHub Releases に
   自動公開するワークフローを設計してください。
   チェンジログ、ダウンロードリンク、引用形式も自動生成してください。
   ```

6. **コードレビュー・品質メトリクスの可視化**
   ```
   GitHub Actions で実行されたテスト結果、カバレッジ、複雑度、静的解析結果を
   README や Wiki に自動で集約・表示するダッシュボードを作成してください。
   ```

7. **GitHub Discussions での知識共有**
   ```
   FAQ、実装パターン、トラブルシューティング情報を GitHub Discussions で
   構造化して共有するテンプレートとガイドラインを作成してください。
   ```

8. **リリース準備・チェンジログ自動生成**
   ```
   マージされた PR や Commit メッセージから、自動的にチェンジログを生成し、
   リリースノート、マイグレーション手順、既知の問題を含むドキュメントを作成してください。
   ```

9. **マルチリポジトリのコード跨域検索**
   ```
   CoreClaw 本体、複数のスキルリポジトリ、MCP サーバーリポジトリをまたがる
   コード検索・依存関係追跡機能を実装するシステムを設計してください。
   ```

10. **リポジトリアーカイブ・長期保管戦略**
    ```
    完了した実験プロジェクトのリポジトリをアーカイブ・バージョニングし、
    長期保管・監査への対応を可能にするワークフローを設計してください。
    メタデータ記録、アクセス制限を含めてください。
    ```

## 6. Data Analysis & Visualization（データ分析・可視化）

図表作成、エクスプロレーティブデータ分析、レポート生成。

1. **多変量解析の一括実行・可視化**
   ```
   複数の患者データセット（臨床検査値、画像定量値）を入力して、
   相関分析、主成分分析、クラスタリング、生存分析を一括実行し、
   結果を Publication-ready な図表と統計レポートで出力してください。
   ```

2. **インタラクティブダッシュボードの生成**
   ```
   実験データと統計結果をもとに、Plotly / Dash を使ったインタラクティブダッシュボードを
   自動生成してください。フィルタリング、ドリルダウン、リアルタイム更新を支援してください。
   ```

3. **異なるデータソースの統合ビジュアライゼーション**
   ```
   複数の実験から得たデータ（CSV、JSON、DB クエリ）を統合して、
   共通指標で比較可能な図表・テーブルを生成するパイプラインを実装してください。
   ```

4. **外れ値検出と品質フラグ付与**
   ```
   測定データセットにおいて、統計的異常値、前提条件逸脱、
   機器キャリブレーション異常などを自動検出し、
   ```
